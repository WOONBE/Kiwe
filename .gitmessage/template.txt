# 🐭 Subject RulePermalink
# 1. 제목은 최대 50글자 넘지 않기
# 2. 마침표 및 특수기호 사용x
# 3. 첫 글자 대문자, 명령문 사용
# 4. 개조식 구문으로 작성(간결하고 요점적인 서술)

# 🐭 Body RulePermalink
# 1. 한 줄당 72자 내로 작성
# 2. 최대한 상세히 작성
# 3. 어떻게 보다는 '무엇을', '왜' 변경했는지에 대해 작성

# 🐮 Footer RulePermalink
# 1. 유형: #이슈 번호의 형식으로 작성
# 2. 이슈 트래커 ID를 작성
# 3. 여러개의 이슈 번호는 ,로 구분
# 4. 이슈 트래커 유형은 아래와 같다

# Issue Tracker	설명
# Fixes:	이슈 수정중(아직 해결되지 않은 경우)
# Resolves:	이슈를 해결한 경우
# Ref:	참조할 이슈가 있을 때 사용
# Related to:	해당 커밋에 관련된 이슈 번호(아직 해결되지 않은 경우)
##################
#✨feat - Jira Issue 번호 : 새로운 기능 추가 (로그인 기능, 새로운 API 추가)
#🐛fix - Jira Issue 번호 : 버그 수정 (NullPointerException 해결)
#🚑fix - Jira Issue 번호 : 긴급 핫픽스 (서비스 장애 복구, 긴급 배포)
#🧪test - Jira Issue 번호 : 테스트 코드 추가 또는 수정 (유닛 테스트 추가, 테스트 개선)
#♻️refactor - Jira Issue 번호 : 기능 변경 없이 코드 구조 개선 (함수 분리, 코드 중복 제거)
#✏️style - Jira Issue 번호 : 단순 오타 수정
#🎨style - Jira Issue 번호 : 코드 포맷팅스타일 수정 (기능 변경 없음)(들여쓰기 수정, 공백 제거)
#💄style - Jira Issue 번호 : UI 스타일 추가 or 변경 (UI 구현 및 색상 변경, 버튼 크기 수정)
#🔥chore - Jira Issue 번호 : 불필요한 코드파일 삭제 (주석 제거, 사용하지 않는 파일 삭제)
#📦chore - Jira Issue 번호 : 패키지 추가 또는 의존성 설정 (라이브러리 추가 또는 업데이트)
#🔧chore - Jira Issue 번호 : 설정 파일 수정 (Gradle 설정 수정)
#🚀release - Jira Issue 번호 : 배포 또는 릴리즈 준비 (배포 브랜치 병합, 버전 태그 추가)
#🔼deps - Jira Issue 번호 : 의존성 업그레이드 (Retrofit 버전 업그레이드)
#🔽deps - Jira Issue 번호 : 의존성 다운그레이드 (호환성 문제로 버전 낮추기)
#⚡perf - Jira Issue 번호 : 성능 최적화 (API 호출 최적화, 로딩 시간 단축)
#🐎perf - Jira Issue 번호 : 실행 속도 개선 (데이터 처리 속도 개선)
#📝docs - Jira Issue 번호 : 문서 작성 또는 수정 (README 수정, 주석 추가)
#🚨chore - Jira Issue 번호 : 린터 경고 또는 오류 수정 (ESLint, Checkstyle 경고 해결)
#🔒security - Jira Issue 번호 : 보안 패치 또는 개선 (취약점 수정, 인증 추가)
#🏗️refactor - Jira Issue 번호 : 큰 구조 변경 (모듈화 작업, 폴더 구조 변경)
